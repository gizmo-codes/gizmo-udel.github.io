Toner Project

My largest and most practical project while learning Java was an inventory management system. The project gave a real world scenario of a client coming to our classroom and proposing a problem they had that we were then expected to find a solution for.

The project was very enlightening and gave me a real look at the process that takes place when you're working with a client on a product that they want. This project was done in a group as most projects done in field are conducted in teams.

My team had to conduct the following:

  • Requirements Interview

    • We interviewed the client with intelligent questions to figure out exactly what he wants from our team. After a discussion within the team we can then plan follow-up questions the team is unsure of or needs clarification on. We then moved into documentation.

  • Documentation

    • By now our team had the appropriate requirements needed for the project and we were able to move onto documentation. Here we produced a full software requirements specification document (SRS) which then allowed us to flow into a unified modeling language (UML) diagram. From here we started implementation.

  • Implementation

    • Here is where we finally get into the programming. Now that we have our proper documentation and general idea for the application we can then start writing code. The documentation made this much easier as it is laid out right in the paper what inputs, outputs, functions, etc. are needed. The program practically writes itself at this point.







Printer Cartridge Inventory System (PCIS)

We named our project Printer Cartridge Inventory System. Our goal was to make it easier for the tech support workers at our location to check if there was ink available for the printers located within their institution.

This is our simple interface.

Here, the user can enter a barcode found on the side of the printer to check the current available inventory for matching ink cartridges.

If the user enters an incorrect number that is not in our database, there is a prompt presented with feedback.

Once a correct code is entered that corresponds to an existing printer, the model number of the printer (B5460DN) and ink model number that corresponds to that printer (X5GDJ) are displayed with the current stock available.

This lets the technician know if there is ink available for replacement without having to physically check and also gives an idea as to weather they need to order more as the current stock may be getting low.

Finally, if the technician wants to enter a comment for the room so future or repeat issues can be corrected quickly, then they can enter a comment into the box shown.

Multiple comments can be entered for the same room and have a relevant information tagged above each comment such as the date and time that the comment was made.
